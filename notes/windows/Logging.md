## Логи Windows

Логи Windows зберігаються у бінарному форматі в папці `C:\Windows\System32\winevt\Logs`.  
EVTX — формат файлів логів.  
Тільки для Security логів існує понад 500 Event ID, а загалом — тисячі різних подій.

### Security log

Серед усіх логів Windows, увімкнених за замовчуванням, Security event log приносить найбільше користі.  
Розгляну два найважливіші Security логи:

|**Event ID**|**Призначення**|**Логування**|**Обмеження**|
|---|---|---|---|
|**4624** (Successful Logon)|Виявлення підозрілих/мережевих логінів та точки початку атаки|Логується на машині, до якої здійснюється доступ|**Шумний**. На завантажених серверах — сотні подій на хвилину|
|**4625** (Failed Logon)|Виявлення брутфорсу, password spraying чи сканування на вразливості|Логується на машині, до якої здійснюється доступ|**Непослідовний**. Багато нюансів, які можуть призвести до неправильного розуміння події|

### Керування користувачами

|**Event ID**|**Опис**|**Зловмисне використання**|
|---|---|---|
|**4720** / **4722** / **4738**|Облікового запису користувача було створено / увімкнено / змінено|Атакуючі можуть створити backdoor-акаунт або увімкнути старий, щоб уникнути виявлення|
|**4725** / **4726**|Облікового запису користувача було вимкнено / видалено|У деяких складних випадках зловмисники можуть вимикати привілейовані акаунти, щоб сповільнити свої дії|
|**4723** / **4724**|Користувач змінив свій пароль / Пароль користувача було скинуто|Маючи достатні права, зловмисники можуть скинути пароль і отримати доступ до потрібного користувача|
|**4732** / **4733**|Користувача додано до / видалено з групи безпеки|Атакуючі часто додають свої backdoor-акаунти до привілейованих груп, наприклад "Administrators"|

Корисно копіювати LogonID (ID сесії логону користувача), щоб знаходити інші дії того самого користувача.

### Моніторинг процесів

|**Event Code**|**Призначення**|**Обмеження**|
|---|---|---|
|**4688** (Security Log: Process Creation)|Логує кожен запуск нового процесу, включно з командним рядком і даними про батьківський процес|Вимкнено за замовчуванням, потрібно вмикати|
|**1** (Sysmon: Process Creation)|Замінює 4688 і надає більш просунуті поля, наприклад хеш процесу та його підпис|Sysmon — зовнішній інструмент, не встановлений за замовчуванням|

|**Event ID**|**Альтернатива в Security log**|**Призначення події**|
|---|---|---|
|**11 / 13** (File Create / Registry Value Set)|**4656** для змін файлів та **4657** для змін реєстру, обидва вимкнені за замовчуванням|Виявлення файлів, скинутих malware, або змін у реєстрі (наприклад, для persistence)|
|**3 / 22** (Network Connection / DNS Query)|Прямої альтернативи немає, потрібна додаткова конфігурація firewall та DNS|Виявлення трафіку від недовірених процесів або до відомих шкідливих адрес|

### Логування команд

Для трекінгу PowerShell команд можна дивитись файл `C:\Users\<USER>\AppData\Roaming\Microsoft\Windows\PowerShell\PSReadline\ConsoleHost_history.txt`, але є багато способів сховати активність від цього файлу, тому краще увімкнути **PowerShell ScriptBlock Logging** і дивитись логи **4104**.
